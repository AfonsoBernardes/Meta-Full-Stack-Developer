=== Core Internet Technologies ===

--- Internet Protocols ---

* The way computers send and recieve data across the internet can compared to a postal system. Computers have addresses we can use to send packets of information, these are called IP Adresses.

* When we send data between computers, the computers are the destination which request and recieve data, and the network are the routes the information travels across. Internet protocols are what make these transactions possible, IP Version 4 and IP Version 6 are the most widely use standards.

* Every computer on a netwrok is assigned an IP Address:

    > In IP Version 4, an IP address contains four octets, separated by periods, i.e., 192.0.2.235.

    > In IP Version 6, an IP address contains eight groups of hexadecimal digits separated by a colon.

* When we send data across a netwrok, we send it as a series of messages known as IP Packets, or dataframs, which contain a header and a payload, which is the data. These packets include the destination IP address and the source IP address in the headers.

* When sending multiple messages, it is possible they are recieved out of order, damaged or even lost. The payload contains other protocols:

    > TCP (Transmission Control Protocol) can solve all of the three issues but at a small delay. It is used to sen data which must arrive correctly and in order, like text or image files.

    > UDP (User Datagram Protocol) solves the corrupted package issue, but packets can still arrive out of order or not at all; it is used for sending data which can tolerate some loss, like voice calls or video streaming.

--- Hypertext Transfer Protocol (HTTP) ---

* Core request-response operation protocol used across the web, enabling the browsers to communicate with web servers hosting websites. A client sends an HTTP request to a server which sends a HTTP response back to the client.

* A HTTP request contains a method, path protocol and headers

    <METHOD> <PATH> <PROTOCOL VERSION>
    <HEADERS>
    \r\n
    <REQUEST BODY>

    GET / HTTP/1.1
    Host: developer.mozilla.org
    Accept-Language: en

    {"key": "value"}

    > METHOD: describes the type of action to perform (eg. GET, POST, PUT, DELETE)
    > PATH: where the resource is stored on the web server (eg example.com/images)
    > HEADERS: contains additional information about the request and the client
    > BODY: included when using the HTTP POST and PUT methods to transmit data.

* A HTTP response is similar to the request

    <PROTOCOL> <STATUS CODE> <MESSAGE>
    <HEADERS>
    \r\n
    <MESSAGE BODY>

* Status codes are grouped according to their purpose:

    > Informational are provisional and interim before the actual response. Range from 100 to 199.
    > Successful indicate the request was successefully processed by the server. Range from 200 to 299.
    > Redirection indicate to the client that the resource was moved to a different part. Range from 300 to 399.
    > Client Error indicate the request contains bad syntax or content and cannot be processed by the server. Range from 400 to 499.
    > Server Error indicate that a failure occured on the web server when processing the request. Range from 500 to 599.

* HTTPS uses encryption to establish a secure connection, maintaining simple HTTP request and reponse behaviour; before the content is sent, it is turned into a "secret code" which can only be read by the destination computer.


--- Intro to HTML, CSS & JavaScript ---

* These three technologies "work together" to create a functional webpage. HTML defines the contents of the page, CSS adds positioning and styling while JavaScript manages the logic behind it.

* The .html file references the .css file to apply styling and the .js file. The browser will retrieve the .js file and process it. In a video player webpage with a button to play/ pause a video, the .js file will
    > registers a listener on the button which executes some code when clicked.
    > when the code runs, it checks the current state of the vidoe, i.e., if the video is paused, it will play the video and change the icon on the button to pause; on the other hand, if the video is playing, the .js code will pause the video and change the icon to a play icon.


--- Other Internet Protocols ---

-> Dynamic Host Configuration Protocol (DHCP)

    * When a computer connects to a network, the DHCP  is used to assign your computer an IP address.

    *A computer communicates over User Datagram Protocol (UDP) using the protocol with a type of server called a DHCP server. The server keeps track of computers on the network and their IP addresses. It will assign your computer an IP address and respond over the protocol to let it know which IP address to use.

    *Once your computer has an IP address, it can communicate with other computers on the network.


-> Domain Name System Protocol (DNS)

    * A computer needs a way to know with which IP address to communicate when you visit a website in your web browser. The Domain Name System Protocol, provides this function.
    * The computer then checks with the DNS server associated with the domain name and then returns the correct IP address.


-> Internet Message Access Protocol (IMAP)

    * A device (mobile or tablet) needs a way to download emails and manage the mailbox on the server storing the emails; whihc is the purpose of this protocol.

-> Simple Mail Transfer Protocol (SMTP)

    * Once the emails are on the device, one needs a way to send emails. The Simple Mail Transfer Protocol allows email clients to submit emails for sending via an SMTP server. One can also use it to receive emails from an email client, but IMAP is more commonly used.


-> File Transfer Protocol (FTP)

    * Running websites and web applications on the Internet needs a way to transfer the files from the local computer to the server they'll run on. FTP is the standar protocol used for this purpose.

    * FTP allows you to list, send, receive and delete files on a server. Your server must run an FTP Server and you will need an FTP Client on your local machine.


-> Secure Shell Protocol (SSH)

    * When working with servers, we need a way to log in and interact with the computer remotely; the most common method being Secure Shell Protocol.

    * Using an SSH client allows you to connect to an SSH server running on a server to perform commands on the remote computer.

    *All data sent over SSH is encrypted. This means that third parties cannot understand the data transmitted. Only the sending and receiving computers can understand the data.


-> Secure File Transfer Protocol (SFTP)

    * The data is transmitted insecurely when using the File Transfer Protocol. This means that third parties may understand the data that you are sending. This is not right if you transmit company files such as software and databases.

    * The SSH File Transfer Protocol, alternatively called the Secure File Transfer Protocol, can be used to transfer files over the SSH protocol. This ensures that the data is transmitted securely. Most FTP clients also support the SFTP protocol.

* A typical webpage is a single pages with HTML, CSS and JS. A website is a collection of webpages under the same domain name.

* Website and Web Applications are often used interchangibily, whoever, the key difference is the level of interactivity and dynamic content.
